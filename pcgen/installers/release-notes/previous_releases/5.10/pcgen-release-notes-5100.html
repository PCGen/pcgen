What's new for v5.10.0

A lot of things happened since the 5.8 release. 282 Feature requests (121 code, 74 data, 8 tags, 68 docs, 11 output sheets) 396 Bug fixes (196 code, 153 data, 29 docs, 15 output sheets, 3 other) and 49 New sources (26 prod, 23 alpha). Here's the summary, see the Change Log for the details.

Main Data Changes

We have completed support for the core datasets with the addition of the Epic and Divine subsets in the RSRD and the completion of the Future and Arcana subsets of the MSRD.

New code enhancements have allowed us to create Default Monster Kits for most of the monsters in our repository. The RSRD and MSRD monsters have received close attention with an extensive sweep for data errors. The Default Monster Kits for these sets include weapons, armor and other default equipment. A new output sheet, csheet_fantasy_statblock_SRD.htm, has been added which is formatted to look as close to the text of the SRD as PCGen can for the purpose of making the checking of Default Monster Kits easier.

We have added support for the Vitality and Wound point system presented in Unearthed Arcana, an alternative to the standard Hit Point system. You can switch to this system from the Preferences > House Rules window and the output sheet will then list Vitality and Wound points instead of Hit Points.

Since the inception of the program we have mistakenly named the Will check Willpower. This has been corrected in 5.10. Users with homebrew datasets will need to update the Will check bonuses in their sets to use Will instead of Willpower. Several other objects have also received some name changes as well. In the MSRD the class named "Battlemind" was corrected to "Battle Mind", the class "Archaic Weaponmaster" was corrected to "Archaic Weaponsmaster". In the RSRD the class "Warmind" was corrected to "War Mind".

We have added many Wildshape templates which allow a Druid with to assume a Wildshape form and have the altered stats reflected in the output sheet.

We have added temporary bonus templates for Two Weapon Defense & Fighting Defensively which can be applied to characters using these actions and the bonuses gained will be reflected on the OS. Other similar conditional templates such as Cowering, Energy Drained and Fatigued have also been added.

The RSRD Lycanthrope Templates have received several rounds of improvements this cycle. Applying one is almost a one step process as the templates will now add temporary animal hit dice (which is removed when the template is removed). All that´s left to do is spend the skill points gained from the animal hit dice.

Random Treasure Table kits have been added to the RSRD set. A variety of these sets enable the addition of randomly selected items which can range from a kit that adds a particular type of item (like an Art object) to kits that roll on a series of tables based on the characters CR. These kits are an implementation of the treasure tables in the RSRD.

A Blank data set with commented out examples has been added to the data/my_dataset folder. This is designed to provide a basic file structure for a beginner to start creating their own dataset.

GUI improvements: Common interface elements

Many tabs in PCGen now have common interface elements to aid in finding and sorting through the lists.

The Filter field allows you to search for items by any text you enter, PCGen will display only those items which have names or types that match the text entered, this is the reason the Kukri is displayed when filtering for the word "dagger", it has the "Dagger" type. If you have applied any other filters by using the Filter tool the text filter will limit it´s search to those not filtered out by the filter tool. In other words the items displayed will match both the Filter tool and the text Filter.

On the right side of many of the lists column label row is a small icon with three dots in it . When clicked this will produce a pop up menu displaying all the possible columns the list can display with a check mark beside each item currently enabled. Selecting any item will change it´s state from either being displayed or hidden.

New sub-tabs

The Race tab is now divided into two full sub-tabs: Races and Templates.

The Spells Tab is divided into three sub-tabs: Known Spells, Prepared Spells and Spell Books. The Known Spells sub-tab is used to add spells to the PC´s Known Spells lists. A spell caster who must prepare spell ahead of time can use the Prepared Spells sub-tab to create Prepared Spell lists. A spell caster who must record spells in a spellbook can use the Spell Books sub-tab to track which spells are kept in which books.

Other changes

The equipment customizer has been adjusted so that the default name is composed better. For example if you create a Returning Dagger the default name will be "Dagger +1 (Returning)" instead of "Dagger (+1 enhancement to Weapon or Ammunition/Returning)".

A list of things to be done for a character is now displayed on the summary page, and the relevant tabs are flagged. This covers things like feats or languages that need to be selected as well as having unspent skill points etc.

The custom items are now split by game mode, so you'll no longer see your tough's customized plasma rifles in the equipment list when selecting your wizard's dungeoneering gear.

The way PCGen handles weapons which have both martial and exotic proficiencies has been overhauled and feats like weapon focus in them work in a more predictable manner

Éric "Space Monkey" Beaudoin for the PCGEN Team -- April 27, 2006.

SRD, RSRD and MSRD Directory reorganization

In the 5.9.7 release we have reorganized the core datasets directories. Essentially we have moved the files loaded by each Partial set into it's own directory. We have done this to simplify maintenance and to stream line their use, since each is now a self-contained dataset you can be confident that no file is double loaded when you mix and match Partial sets. You can also load any combination of sets and know that only the basics set is required (with the exception of the Epic Psionics set which also requires the Psionics set).

Impact: If you have created custom datasets which use relative links to load files from the core datasets these links will be broken. You will need to check out the new structure to see how it need to be fixed. Similarly if you have created a custom pcc with the Customize button in the Sources Tab you will need to manually edit it to correct the paths or simply recreate it.

For the SRD and RSRD this change is not expected to break any previously created characters since the dataset names were not changed.

Changes to the MSRD on the other hand are going deeper as they involve a major overhaul of the PCC Type and Campaign tags. Where the complete set previously only loaded the Basics and Menaces sections of the MSRD, there is now a much more modular structure that will help to better cope with the multitude of genres that can be covered by the Modern game.

The new setup now shows under Complete: MSRD Arcana, MSRD Basics, MSRD Future, and MSRD Menaces. These sets will each load a complete section, as those appear as partially downloadable sections of the MSRD on the WotC webpage. To exactly do that, the section previously called "Core" has been renamed to "Basics" as that is what the MSRD uses. Of course there are still the optional partial sets for users that want to load only certain components, like loading the Basics without FX.

But all those changes to the MSRD come at a price. Pre-existing characters will most certainly be broken and are unlikely to still work with this new version. To work around this you can try the following steps:

  • 1st-make a copy of your character's .PCG file, in case you might mess up with editing the original;
  • 2nd-open the original .PCG file in a text-editor;
  • 3rd-delete the CAMPAIGN:xxx tags in the file (i.e. CAMPAIGN:Modern);
  • 4th-save the file again;
  • 5th-start PCGen;
  • 6th-select and load all sources that your character has previously taken choices from;
  • 7th-load the character;
  • 8th-if the character now loaded successfully, save it in the new format. That will store which sources were loaded with it, so you can treat that file as normal in the future.

This procedure should usually work.

Installation

Prerequisites

Java 1.4.x or Java 5. PCGEN 5.10.0 is not compatible with Java 1.3.x and it is NOT SUPPORTED. To get the latest version of Java, go to java.com.

Windows Users

We recommend that you use the Windows installer.

Mac Users

We recommend that you use the OS X DMG Mac Installer, it is now working with OS X 10.4 (Tiger). This has been a problem in the past due to a change on Apple’s part in how the menu bar was handled with java 1.4 on Tiger, a packaged java app set to use the main window instead of the application window caused the program to crash at startup. At this point in time the solution is to set the menu bar to appear in application menu.

Also new in this version is the configuration of the program, all the necessary files are now included with the program package. You can still access the files by right clicking the program and choosing Show Package Contents. See the notes included in the installer DMG for all the details.

Others (or advanced) users

Pick the package with the features you want and unzip it. It is highly recommended that you use a new directory for every new release. Once the package is unzipped, see docs\index.html for the full documentation.

Alpha Data Sets

The Alpha directory that contains data sets that were not judged to be "production" grade, is included in a separate package. They vary from "better then nothing" to "nearly ready for production".

Read the doc!

Please make sure you read the docs to familiarize yourself with the changes. You can access it from docs/index.html located in the PCGen directory.
