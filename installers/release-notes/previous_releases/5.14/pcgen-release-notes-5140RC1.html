What's new for v5.14.0 RC1

PCGen v5.14.0 RC1 is the first release candidate for PCGen 5.14. It represents a product we believe is ready for production use and which we would like your assistance in testing out. In particular the PCGen team would appreciate your feedback on importing your characters and data from earlier versions of PCGen. The highlights of the changes for PCGen 5.14 are listed below:

Major Changes:
  • User programmable preview tab
  • Move point buy definitions to Custom data
  • Data installer for easy installation of out of cycle data releases
  • Add cover art, logos, publisher links and purchase links for many source books
  • Extensive tidy-up of the LST syntax in preparation for future automatic migration
  • Convert class and racial abilities to ability objects improving visibility and editability
New Output Sheets:
  • Graphical preview sheet.
  • Preview sheet with rolling functionality
New Sources:

    Character Changes

    We've been making some major changes behind the scenes to the way special abilities for characters are handled. The benefit is that it lets us model them more accurately, with fighter feats tracked separately to normal feats for example, and that we can output them more accurately on the character sheet or stat block. However there is a downside - this means that there is some migration required for some characters. For details of what you need to do, see the sections for the affected game modes below.

    This revamp of abilities is something we would like your feedback on. So please have a play with them and send your thoughts to the PCGen list.

    Feat Pool Issues

    Many classes grant bonus feats, one of the major advancements in 5.14 is that these feats are now selected in a separate pool from general feats. Instead of a pop up appearing when the character is leveled up a new Feat Pool is created on the Feats subtab from which those feats can be chosen. A side effect of this new feature is that some previously created characters will have an incorrect number of feat slots when loaded in 5.14 for the first time, this is because they now have an empty pool for their bonus feats even though they have already previously been selected and put into the general feat pool. This must be corrected manually because PCGen has no way to know which feats were selected as bonus feats.

    To fix this problem, you can take the following steps:

    1. Determine which Feats in the standard Feat Pool were chosen as Bonus Feats.
    2. Remove those Feats from the standard Pool and change the number in the Feats Remaining field to 0.
    3. Select the Bonus Feat Pool and add your Feats back.

    Fantasy Character Changes

    When a pre-generated SRD (3rd ed) or RSRD (3.5 ed) ranger, archmage or high level rogue character is loaded, it will not recognize the selected abilities anymore. For that reason one or more error messages will be displayed at character load, which read: "Warning: Unable to Find Ability: xxx", where xxx is the name of an ability that the character had previously selected.

    To fix this problem, you can take the following steps:

    1. Write down the abilities that are mentioned in those error messages somewhere.
    2. Go to the Feats tab and select the new "Class Abilities" subtab.
    3. In the lower right corner of the Class Abilities tab you see a drop down menu which reads "Rogue Special Abilities". Select the appropriate type of ability for one of the abilities that you have noted. The "remaining" field should now show a positive number.
    4. In the upper left field only abilities that belong to that group will be displayed. Select your ability(s) for that group there.
    5. Repeat for any abilities that were taken from another group.

    Monk Character Changes

    The 3.5 Monk present a special problem. The Monk gets to choose specific bonus feats at levels 1, 2 and 6 and the Monk can ignore the prerequisites of these feats. As of version 5.14.0 RC1 these selections are made in the Feat tab but previously they were selected from a pop up menu at level up. The problem is that these feat selections are virtual feats (since the Monk ignores the prerequisites) and there is no way to remove previous selections made for pre-existing characters from within the program. There are two options for dealing with this issue: You can simply ignore the new feature and retain the previous selections, or you can edit the character file with a text editor and delete the selections manually. A character with levels of Monk will have a feat pool named "Monk Feat" with 1, 2 or 3 points in it (if no selections have been made), previously generated characters will still retain their selections and those can be seen in the feats tab as virtual feats. If you do not wish to make changes you can set the pool to 0 and simply ignore it. if you wish to delete the previous selections so that you can choose them through the new feature you will need to manually edit your characters .pcg file with a text editor, always do this to a copy of the file in case a mistake is made.

    To edit the Monk:

    1. Open a copy of the character in a text editor.
    2. Scroll down and find the text # Character Abilities
    3. Under this header find the lines which start with: ABILITY:FEAT|TYPE:VIRTUAL|CATEGORY:FEAT|KEY:
    4. After "Key" will be the name of the feat, there are only six options: Improved Grapple, Stunning Fist, Combat Reflexes, Deflect Arrows, Improved Disarm and Improved Trip. Delete these lines.
    5. Save the file and open in PCGen.
    6. Go to the Feat tab and make your selections for the Monk Feat pool, these selections will then appear in the general feat pool as virtual feats.
    7. Save the character.

    MSRD Character Changes

    When a pre-generated modern character is loaded, it will not recognize previously selected Talents, Wealth Rolls, Occupations and it's associated selections anymore. For that reason one or more error messages will be displayed at character load, which read: "Warning: Unable to Find Ability: xxx", where xxx is the name of a Ability that the character had previously selected.

    To fix this problem, you can take the following steps:

    1. Write down the Abilities that are mentioned in the error messages.
    2. The Occupation feature is activated via a template that is added when adding a level of any base class, to get this template applied to a pre-generated character you must add a level of a base class. You can add a level and then remove it as well as remove a level and add it back if you need to stay at the current level (remember to write down your current experience point value before you level up so you can reset it when leveling back down). If you have more than one base class you will get a pop up chooser window for a template with choices like "FastSkills" or "SmartSkills", choose the template that corresponds to the class taken at level 1.
    3. Go to the Feats tab and select the new "Background" subtab.
    4. For most characters you will see an available "Pool" for Wealth and Occupation.
    5. Select the Wealth Pool and choose a Wealth Roll.
    6. Select the Occupation Pool and choose an Occupation.
    7. Once an Occupation is chosen a new pool for that Occupations skill selections becomes available.
    8. Some Occupations also grant a choice of Feats, these options can be found and selected in the "Feats" subtab.
    9. Previously selected Occupation Feats will be in the main Feat Pool, see notes below on how to fix this.
    10. Next select the new "Talents" subtab.
    11. In the lower right corner of the Talents tab you see a drop down menus which reads "Talents". Select a class that one of the talents you have noted was taken from. The "remaining" field should now show a positive number.
    12. In the upper left field only Talents that belong to that class will be displayed. Select your Talent(s) for that class there.
    13. Repeat for any Talents that were taken from another class.

    The way that the Base classes bonus feats are handled has been changed as well. Instead of a pop up appearing when the character is leveled up a new Feat Pool is created on the Feats subtab from which those feats can be chosen. Pre-generated characters will now have an empty pool for these feats even though they have already previously been selected.

    To fix this problem, you can take the following steps:

    1. Determine which Feats in the standard Feat Pool were chosen as Bonus Feats or Occupation Feats.
    2. Remove those Feats from the standard Pool and change the number in the Feats Remaining field to 0.
    3. Select the Bonus Feat Pool and add your Feats back.

    GMGen Known Issue

    There is a known issue with GMGen when used on a Macintosh. In the Initiative Tab when running a combat when you select any character other than the one whose turn it is and click the Damage or Heal buttons the character whose turn it is becomes selected and the damage is applied to him instead of the character which had been selected.

    A solution has not yet been found but fortunately there is a simple work-around. The problem will only manifest when using the System Look and Feel, you can change the Look and Feel by going to Preferences/Appearance/Look and Feel and changing it to Java before activating GMGen and then it will work as it should.

    Installation

    Prerequisites

    Java 5. PCGEN 5.14.0 RC1 is not compatible with Java 1.4.x and it is NOT SUPPORTED. To get the latest version of Java, go to java.com.

    Important Warning

    If you have custom data created using either the LST Editor, or the my_dataset example files, it is very important that you read the Custom Data Sets section below before installing your new version of PCGen.

    Windows Users

    We recommend that you use the Windows installer.

    Mac Users

    We recommend that you use the OS X DMG Mac Installer, it is now working with OS X 10.4 (Tiger). This has been a problem in the past due to a change on Apple’s part in how the menu bar was handled with java 1.4 on Tiger, a packaged java app set to use the main window instead of the application window caused the program to crash at startup. At this point in time the solution is to set the menu bar to appear in application menu.

    Also new in this version is the configuration of the program, all the necessary files are now included with the program package. You can still access the files by right clicking the program and choosing Show Package Contents. See the notes included in the installer DMG for all the details.

    Others (or advanced) users

    Pick the package with the features you want and unzip it. It is highly recommended that you use a new directory for every new release. Once the package is unzipped, see docs\index.html for the full documentation.

    Custom Data Sets

    PCGen provides two simple ways of adding your own data, these are the inbuilt LST Editors and the starter data set my_dataset. Both of these need some attention when upgrading PCGen.

    Migrating Data Created in the LST Editor

    If, as is recommended, you install PCGen into a new directory each time you upgrade, you will need to copy over the data created using the LST Editors. To do this follow these steps.

    1. Stop PCGen, if it is currently running. PCGen must not be running when you copy the files, as PCGen writes a new copy of these files on exit.
    2. Copy the data/customsources folder and all of its contents from your old PCGen install directory to the new install directory.
    3. Start the new version of PCGen and verify that your custom data is present.

    If you install PCGen in the same directory as your previous install, your LST editor data will still be present in the new version.

    Migrating Data Created using the my_dataset files

    PCGen ships with an example set of data files ready to go to create you own data. This is in the data/my_dataset folder. As the example files ship with PCGen, it is very important to note that these can be overwritten by a new PCGen install if you have not renamed the data set and install the new version of PCGen in the same location as the old one.

    As a result, we strongly recommend that when using the my_dataset example files, you create a copy in a new folder and work with that copy, rather than editing the example files directly.

    If you install PCGen into a new directory each time you upgrade, you may need to copy over the data you have created. If you cannot see your custom data, simply copy over the folder with your custom data to the same location in the new PCGen install. The data should then be visible when you restart PCGen.

    If you are installing PCGen into the same directory, backup your custom data first, then make sure it does not use the default my_dataset folder name. Once you are sure you data is safe, then proceed with the upgrade of PCGen.
